# 架构设计文档

## 🏗️ 系统架构概述

### 设计理念
本系统采用"一主四辅"分布式架构，核心思想是**职责分离**和**专业分工**。每个Agent专注于单一职责，通过HTTP REST API进行通信，实现高内聚、低耦合的系统设计。

### 架构优势
1. **可扩展性**：每个Agent可独立扩展和升级
2. **容错性**：单个Agent故障不影响整体系统
3. **可维护性**：代码结构清晰，易于理解和维护
4. **灵活性**：可灵活调整策略和参数

##### 均线粘合算法
```python
def check_ma_convergence(df_2025, ma_periods=[5, 10, 20, 30]):
    """
    检查2025年均线粘合度
    粘合标准：各均线之间距离 < 股价的2%
    """
    ma_values = []
    for period in ma_periods:
        ma = df_2025['close'].rolling(window=period).mean()
        ma_values.append(ma[-20:])  # 取最后20个交易日
    
    # 计算均线标准差
    ma_matrix = np.array(ma_values)
    ma_std = np.std(ma_matrix, axis=0).mean()
    
    # 粘合判断
    current_price = df_2025['close'].iloc[-1]
    return ma_std < current_price * 0.02
```

##### 均线发散算法
```python
def check_ma_divergence(df_2026, ma_periods=[5, 10, 20, 30]):
    """
    检查2026年均线发散（向上排列）
    发散标准：短周期均线 > 长周期均线，且呈上升趋势
    """
    ma_trends = []
    for period in ma_periods:
        ma = df_2026['close'].rolling(window=period).mean()
        # 计算均线斜率（最近10个交易日）
        slope = np.polyfit(range(10), ma[-10:], 1)[0]
        ma_trends.append(slope)
    
    # 检查均线排列顺序
    ma_values = [df_2026['close'].rolling(window=p).mean().iloc[-1] for p in ma_periods]
    
    # 发散判断：均线向上排列且趋势向上
    is_ascending = all(ma_values[i] > ma_values[i+1] for i in range(len(ma_values)-1))
    is_upward = all(trend > 0 for trend in ma_trends)
    
    return is_ascending and is_upward
```

#### 风险评分算法
```python
def calculate_risk_score(signal):
    """
    计算信号风险评分（0-100分，越低越好）
    评分因素：
    1. 回测胜率（权重40%）
    2. 最大回撤（权重30%）
    3. 夏普比率（权重20%）
    4. 价格因素（权重10%）
    """
    score = 50  # 基础分
    
    # 1. 胜率因素
    win_rate = signal.get('backtest_win_rate', 0.5)
    score -= (win_rate - 0.5) * 40
    
    # 2. 回撤因素
    max_drawdown = signal.get('backtest_max_drawdown', 0.1)
    score += max_drawdown * 100
    
    # 3. 夏普比率
    sharpe_ratio = signal.get('backtest_sharpe_ratio', 0)
    if sharpe_ratio > 0:
        score -= sharpe_ratio * 10
    
    # 4. 价格因素
    price = signal.get('current_price', 0)
    if 10 <= price <= 50:
        score -= 10  # 10-50元风险较低
    elif price > 100:
        score += 20  # 高价股风险较高
    
    return max(0, min(100, score))
```

#### 仓位分配算法
```python
def allocate_position(signal, available_capital, position_rules):
    """
    仓位分配算法
    考虑因素：
    1. 风险评分
    2. 可用资金
    3. 单票上限
    4. 总仓位限制
    """
    risk_score = signal.get('risk_score', 50)
    current_price = signal.get('current_price', 0)
    
    # 根据风险评分调整仓位比例
    base_ratio = position_rules['initial_position']  # 20%
    risk_adjustment = (50 - risk_score) / 100  # 风险越低，仓位越高
    adjusted_ratio = base_ratio * (1 + risk_adjustment)
    
    # 应用上限限制
    max_ratio = position_rules['max_single_position']  # 30%
    final_ratio = min(adjusted_ratio, max_ratio)
    
    # 计算投资金额和股数
    investment = available_capital * final_ratio
    shares = int(investment / current_price / 100) * 100  # 按手取整
    
    return {
        "ratio": final_ratio,
        "investment": investment,
        "shares": shares,
        "buy_price": current_price
    }
```

## 📊 性能考虑

### 计算复杂度
- **数据采集**：O(n)，n为股票数量
- **策略计算**：O(n × m)，m为策略条件数
- **回测验证**：O(n × t)，t为历史交易日数
- **风险计算**：O(k)，k为信号数量

### 内存管理
- **数据缓存**：LRU缓存策略，限制内存使用
- **分批处理**：大数据集分批处理，避免内存溢出
- **垃圾回收**：及时释放不再使用的数据

### 网络优化
- **连接池**：复用HTTP连接，减少握手开销
- **超时设置**：合理设置超时时间，避免长时间阻塞
- **重试机制**：网络异常时自动重试
